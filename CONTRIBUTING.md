# Contributing to Iconpeek

First off, thank you for considering contributing to Iconpeek! It's people like you who make the open-source community such an amazing place to learn, inspire, and create.

## How to Contribute

### 1. Fork the Repository
Click the **Fork** button at the top right of the GitHub page to create a copy of this repository on your GitHub account.

### 2. Clone Your Fork
Clone your fork to your local machine:
```bash
git clone https://github.com/YOUR_USERNAME/iconpeek.git
cd iconpeek
```

### 3. Create a Feature Branch
Create a new branch for your feature or bug fix:
```bash
git checkout -b feature/your-feature-name
```

### 4. Local Development
Install dependencies (if any) and start the local server:
```bash
npm install
npm start
```
Make your changes and ensure they work as expected.

### 5. Run Tests
Before committing, make sure the smoke tests pass:
```bash
npm test
```

### 6. Commit and Push
Commit your changes with a descriptive message and push them to your fork:
```bash
git add .
git commit -m "feat: add awesome new feature"
git push origin feature/your-feature-name
```

### 7. Create a Pull Request
1. Go to the original [iconpeek repository](https://github.com/Rances/iconpeek) on GitHub.
2. You should see a banner suggesting to open a Pull Request from your pushed branch.
3. Click **Compare & pull request**.
4. Describe your changes and submit the PR.

---

## Code of Conduct & Contribution Guidelines
- Ensure your code follows the existing style guidelines.
- Be respectful and constructive in comments and PR discussions.
